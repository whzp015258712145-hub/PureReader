# PureReader

一个高性能的多格式电子书阅读器，专为macOS设计，支持快速启动和多种电子书格式。

## 功能特性

### 🚀 性能优化
- **超快启动**: 应用启动时间优化至0.5秒内
- **即时响应**: 用户交互响应时间控制在200毫秒内
- **智能缓存**: LRU缓存策略，提升重复访问性能
- **内存管理**: 大文件分页加载，防止内存溢出
- **预加载技术**: 智能预加载相邻页面，提供流畅阅读体验

### 📚 多格式支持
- **EPUB**: 完整支持EPUB 2.0/3.0格式
- **PDF**: 原生PDF渲染，支持缩放和导航
- **TXT**: 纯文本文件，支持多种编码格式
- **MOBI**: Amazon Kindle MOBI格式支持
- **AZW3**: Amazon Kindle AZW3格式支持

### 🌐 编码兼容性
- **UTF-8**: 完整Unicode支持
- **GBK**: 简体中文编码支持
- **Big5**: 繁体中文编码支持
- **ISO-8859-1**: 西欧字符编码支持
- **自动检测**: 智能编码检测和错误恢复

### 🎨 阅读体验
- **主题切换**: 日间/夜间模式
- **字体调节**: 字号、行距、边距自定义
- **目录导航**: 快速章节跳转
- **多语言**: 中英文界面支持
- **窗口管理**: 优化的macOS窗口体验

## 快速开始

### 系统要求
- macOS 10.14 或更高版本
- 至少 4GB RAM
- 100MB 可用存储空间

### 安装步骤

1. **克隆项目**
   ```bash
   git clone <repository-url>
   cd purereader
   ```

2. **安装依赖**
   ```bash
   flutter pub get
   ```

3. **运行应用**
   ```bash
   flutter run -d macos
   ```

### 构建发布版本
```bash
flutter build macos --release
```

## 使用说明

### 打开电子书
1. **拖拽文件**: 直接将电子书文件拖拽到应用窗口
2. **文件选择**: 点击工具栏的文件夹图标选择文件
3. **关联打开**: 在Finder中双击支持的电子书文件

### 阅读设置
1. 点击工具栏的文字图标打开设置面板
2. 调整字号、行距、页边距
3. 切换日间/夜间主题
4. 选择界面语言

### 导航功能
- **目录**: 点击菜单图标查看章节目录
- **翻页**: 使用鼠标滚轮或键盘方向键
- **跳转**: 在目录中点击章节快速跳转

## 技术架构

### 核心组件
- **启动优化器**: 并行初始化和延迟加载
- **格式检测器**: 智能文件格式和编码识别
- **解析器工厂**: 多格式解析器统一管理
- **渲染引擎**: 跨格式内容渲染
- **缓存管理器**: LRU缓存和内存优化

### 依赖库
- `flutter_riverpod`: 状态管理
- `epub_view`: EPUB格式支持
- `file_picker`: 文件选择
- `window_manager`: macOS窗口管理
- `app_links`: 文件关联处理
- `shared_preferences`: 设置持久化

## 开发指南

### 项目结构
```
lib/
├── main.dart              # 应用入口和启动优化
├── home_screen.dart       # 主界面和阅读器
├── reader_state.dart      # 状态管理和配置
├── parsers/              # 格式解析器
├── renderers/            # 渲染引擎
├── cache/                # 缓存管理
└── utils/                # 工具类
```

### 添加新格式支持
1. 在 `parsers/` 目录创建新的解析器类
2. 实现 `EbookParser` 接口
3. 在 `EbookParserFactory` 中注册新格式
4. 更新格式检测逻辑

### 性能优化原则
- 延迟加载非关键组件
- 使用异步处理避免UI阻塞
- 实现智能缓存策略
- 优化内存使用 和资源清理

## 故障排除

### 常见问题

**Q: 应用启动缓慢**
A: 检查系统资源使用情况，清理应用缓存，确保有足够的可用内存。

**Q: 文件无法打开**
A: 确认文件格式是否支持，检查文件是否损坏，尝试重新下载文件。

**Q: 中文显示乱码**
A: 在设置中手动选择正确的编码格式，或使用UTF-8编码的文件。

**Q: 内存使用过高**
A: 关闭不需要的书籍，清理应用缓存，重启应用释放内存。
